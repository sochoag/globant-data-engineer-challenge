Globant Data Engineer Challenge

Welcome to the user interface for the RESTful API of the Globant challenge solution.

This web page provides a way to interact with the API, which offers functionalities for data migration, transaction insertion, and data backup/restoration, as described in the challenge.

Main Features

  • Data Migration: Migration of historical data from CSV files to a MySQL database.
  • RESTful API: Interface for the RESTful API service to insert new transactions (up to 1000 records per request).
  • Backup/Restoration: Interface for the backup and restoration functionality of each table in AVRO format.
  • Reports: Generation of reports such as 'quarterly hires' and 'departments above average hires'.
  • Security: Basic authentication to access protected API endpoints.

API Endpoints

The API is built with FastAPI and offers the following endpoint groups:

Data Endpoints

  • POST /departments: Creates a new department.
  • POST /jobs: Creates a new job.
  • POST /employees: Creates a new employee.

Backup/Restore Endpoints (AVRO)

  • GET /avro/backup/departments, /jobs, /employees, /all: Backs up the table.
  • POST /avro/restore/departments, /jobs, /employees: Restores the table.

Report Endpoints

  • GET /report/quarterly_hires: Quarterly hires report.
  • GET /report/departments_above_average: Departments above average hires.

See the API documentation for complete details on request parameters and responses.

Usage

To begin using the API, ensure that the application is running correctly (see the README for instructions on how to run it with Docker).

You can interact with the API endpoints using this web page or any HTTP client (such as curl, Postman, Insomnia, or a web browser). Protected endpoints require basic authentication with the configured credentials.
